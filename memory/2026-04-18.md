### User Search (Find Friends) — Backend Rewrite
- Replaced broken Prisma findMany with raw SQL using $queryRaw
- Ranking: exact username (6) → exact display_name (5) → starts-with username (4) → starts-with name (3) → contains username (2) → contains name (1)
- Fully case-insensitive via LOWER()
- Strips leading @ automatically
- Offset-based pagination (cursor = numeric string offset)
- Tie-break: follower_count DESC, created_at DESC
- Frontend debounce tightened to 220ms

### Find Friends — Search State Persistence
- Back from profile → restores query + results (sessionStorage FF_ACTIVE_KEY)
- Leave to another tab → clears active search, shows recent history instead
- Uses navigatingToProfileRef flag in useEffect cleanup to distinguish the two cases
- Recent search history: stored in localStorage (FF_HISTORY_KEY), max 10 entries
- Each recent entry: tappable (opens profile), × to remove individually, "Clear all" button
- History populated when user taps any search result
