# Memory — April 19, 2026

## Clocked App — Changes Today (April 19)

### Performance / Load Speed
- Added keep-alive cron: backend pinged every 9 min so Railway never cold-starts
- Code splitting via Vite: initial JS bundle dropped from 431KB → 127KB
- All pages lazy-loaded via React.lazy/Suspense — vendor chunk cached after first visit
- Gzip compression added to backend
- Both frontend (railway up) and backend (git push) deployed

### Like/Dislike System — Full Rebuild
- Created `src/lib/postStore.js` — global module-level store, single source of truth for reaction state
- All PostCards subscribe to postStore — liking on any screen instantly syncs all other screens
- `seedPost()` now only runs once on mount (seededRef guard) — never re-seeds during in-flight vote
- Vote lock replaced with queue model: taps are instant visually, requests serialized (no drop, no lag)
- Backend vote routes: now use DB transaction + recalculate counts from postReaction table (no deltas, no drift)
- DB counts were corrupted (likes=3, dislikes=-2) — fixed via fix_counts.js script, recalculated from actual reactions
- fix_counts.js committed to backend repo for future use
- Backend clamp: raw SQL GREATEST(0,...) prevents negative counts at DB level

### Notification Spam Fix
- `notify.js`: new `upsertReactionNotify()` function — one reaction notification per actor per post
- If actor already has a reaction notification on a post, it updates in-place (like→dislike or vice versa)
- If reaction removed, existing notification is deleted
- No more flood of notifications from rapid tapping

### Create Post — Draft Persistence
- Draft auto-saved to sessionStorage on every change (employer, rating, header, body, media previews)
- Draft restored on mount — switching tabs and coming back restores everything
- Draft cleared only on successful submit or edit mode
- Text selection bug fixed: SwipeHandler now bails out immediately if touch starts inside input/textarea

### Create Post — Employer Logo
- BusinessLogo now shown in the selected employer box after picking a company
- Logo on left, name + address to the right, clear button far right — matches search results layout

### Create Post — Rating Selector
- `.rating-option.selected .label` CSS rule added: selected option label turns black (#000000)
- Unselected options remain gray (var(--text-secondary))
- Dark mode fix only, light mode unaffected

### Mobile Header — Three-dot Menu
- `MobileProfileMenu` (three-dot account switcher) removed from all sub-pages
- Was showing on company pages, post detail, find-friends, etc. — now shows nowhere in mobile header
- Profile tab has its own inline three-dot in the hero section (unchanged)
- Changed `!isMainTab && <MobileProfileMenu />` → `location.pathname === '/profile' && <MobileProfileMenu />`

### Comment Sheet — Desktop Parity
- CommentSheet now rendered via `createPortal(document.body)` from PostCard — no more stacking context issues
- Desktop sidebar offset fix: `.comment-sheet-panel` gets `margin-left: 120px` on desktop (≥768px) to center over feed column
- Desktop: `88vh` height, `20px` top border radius, cleaner card feel
- Mobile: unchanged — same slide-up animation, same timing, same layout
- Same component used on both platforms — no separate desktop comments system

## Deployment Notes

- Backend deploy: git push to main (Railway auto-deploys)
- All changes today deployed and confirmed live
