
## Railway Cache Issue (confirmed again 2026-03-30)
- Railway caches `index.html` aggressively — even after `railway up`, old HTML with old JS hashes can persist
- Fix: bump Vite output version in `vite.config.js` (v6→v7 etc.) AND run `railway redeploy --service frontend --yes` after `railway up`
- The redeploy flushes the CDN cache and forces the new index.html to be served
- Always verify with: `curl -s https://frontend-production-d4bd.up.railway.app | grep -o 'assets/index-[^"]*js'`
