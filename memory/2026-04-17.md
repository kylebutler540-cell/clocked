# Memory — April 17, 2026

## Clocked App — Changes Today (April 17)

### Message Thread Back-Swipe / Reload Fix (major architecture change)
- Root cause: `/messages` and `/messages/:userId` were separate route instances — React Router unmounted/remounted the whole component on every navigation, killing cache
- Final fix: lifted `ConversationView` **out of the route tree entirely** — it now renders at the `AppInner` level in App.jsx as a fixed overlay, never unmounts
- `MessagingContext` expanded to hold `activeThreadUserId`, `openThread()`, `closeThread()`
- `Messages` page just syncs route params into context — doesn't render ConversationView at all
- Result: swiping back from profile → DM thread is instant, no remount, no refetch, no flicker — same DOM stays alive

### Message Inbox Performance (batched queries)
- Old: 4 DB queries per conversation (user lookup, unread count, 2 follow checks) = 20+ round trips for 5 convos
- New: 3 batched queries total regardless of inbox size — `findMany` for users, `groupBy` for unread counts, `findMany` for follows
- DM thread open: messages fetch + mark-read now fire in parallel instead of sequential

### ConversationView — Instant Cache on Mount
- `useState` initializer reads localStorage cache synchronously — component renders with data on first paint
- Silent poll only calls `setMessages` if message count actually changed — prevents re-render flash on background polls

### Mobile Comments — Instagram-Style Bottom Sheet
- Mobile only: comments now open as a slide-up bottom sheet overlay (was navigating to separate page)
- Desktop unchanged
- Swipe-to-close only triggers from the handle bar at top — scrolling comments list never accidentally closes
- Post not shown inside drawer (already visible behind overlay)

### Auth / Menu / Icon Fixes
- Signed-out menu: "Add Another Account" now hidden — only Sign Up / Log In shown when not authenticated
- All PostCard interaction handlers (like, dislike, save, comment, flag) verified to redirect to `/signup` when not signed in
- Saved page empty state: replaced 🔐 with purple outline star SVG
- Notifications empty state: purple bell icon instead of muted gray
- All section empty state icons now consistent: `stroke="var(--purple)"` `strokeWidth="1.5"`

### See More Button Fix
- Desktop: inline "See more" at end of truncated text (600 char limit)
- Mobile: inline "See more" at end of truncated text (400 char limit)
- Both use same inline layout — only char limit differs

### Profile Follow Button Flash Fix
- `isFollowing` seeded synchronously from `lsGet('follow_state_${userId}')` at useState init
- No flash of wrong state when opening a profile
