## Anonymous Employer Review App

### Concept
- Anonymous review platform for hourly/trade/service workers to review employers
- Think Glassdoor but unbiased, built for non-office workers
- Social media style (comment, reply, like) — not just static reviews
- Fully anonymous posting

### Business Model (decided)
- Free to post (always, for workers)
- Paywall on reading deep reviews (job seekers pay)
- Companies get free verified accounts (verified via domain email or postcard code — NOT payment)
- No charging companies — prevents fake company rep reviews

### Legal Plan
- Form LLC before launch
- Section 230 protection (platform not publisher)
- Clear ToS — reviews are opinions not facts
- DMCA policy + takedown process
- User flagging → human moderation queue (72hr response commitment)
- Consult startup attorney before launch

### Cold Start Strategy
- Seed with real reviews from team before launch
- Launch in Grand Rapids first, then expand
- Target Reddit (r/antiwork, r/WorkReform) and TikTok for early users
- Make posting frictionless — no account needed for first review

### Build Plan
- I (Kyle/AI) will build the full app using coding agents
- First version: web app (mobile-friendly), not native iOS/Android yet
- Hosting: Railway or Render (~$20/month)
- Domain: ~$15/year
- Timeline: 2-4 weeks for MVP

### UI Design (from sketch #1)

**Rating System**
- 3 emoji faces instead of 1-5 stars
- Red frown = Bad, Yellow flat = Neutral, Green smile = Good
- Selected when creating a post

**Post Card Structure (top to bottom)**
- Anonymous username (e.g. "Anonymous123") + anonymous profile picture
- Place of employment (required — Instagram-style location selector, pulls from database)
- 3 face rating (selected on post)
- Caption — main text where people write their review/experience
- Photo/Video attachment — optional, most posts will be text only
- Bottom actions: 👍 Like | 👎 Dislike | 💬 Comments | ⭐ Starred/Save

**Location / Employer Database**
- Required field — can't post without selecting employer
- Uses Google Maps API (or Apple Maps) as the backend — search and select like Instagram
- No manual pre-loading needed — all businesses already in the map data
- Fallback: if business isn't on the map, user can manually type/add a custom name and proceed
- Launch scope: Grand Rapids area only (geo-restricted at first)

**Moderation**
- Flag button on every post — lets users flag suspicious reviews or ones they know aren't true

### Design & Theme (Q2)
- Dark mode AND light mode toggle — user can switch
- Base colors: black and white
- Accent color: blue — used for logo, bottom nav bar, selected tab states
- Selected tab on bottom nav turns blue

### Vibe (Q3)
- Between Instagram and Reddit — social media feel, not corporate
- Not serious like LinkedIn, but still informative
- Feels like a real platform people want to hang out on

### Content Type (Q5)
- Social feed — posts can be rants, positives, negatives, neutrals
- Focused on insider worker perspective — things they've never had a platform for
- Not totally freeform "post anything" — should be work-related experiences/opinions
- Not structured like a form review either — it's a post with your real words

### Company Profile Page / Filtering (Q6)
- Tap a company → opens company profile page
- Shows all reviews for that location
- Filter by emoji: 😞 / 😐 / 😊
- Search bar to find keywords (pay, management, culture, etc.)
- Topic filter tags (pay, management, culture, etc.) — add more as reviews grow or auto-generate from content

### Visibility (Q7)
- Everything visible to everyone — no paywall on seeing posts
- Paywall is on reading the FULL review text (see monetization below)

### Social / Follow System (Q8)
- Users can follow each other to prioritize their posts in feed
- All accounts anonymous — you only know them by their anonymous username/number
- Follow requests — like Instagram private accounts, user approves or denies
- If approved: can send private messages to get more insider info
- If not approved: message goes to a pending requests inbox on their account
- Following someone = their posts show up higher/first in your feed

### Post Format (Q9 + Q10)
- Posts have an optional bold HEADER at the top (like Reddit post title)
- Header is optional — blank text box at top of post creation, encouraged but not required
- No AI-generated headers — user writes it or leaves it blank
- Below header: their full review text (in-depth, their real words)
- Media: text only, or add up to 10 photos, or a video (cap: ~2 min max)
- Text is most common expected format

### Monetization — Blur Paywall (revised)
- First 1-2 sentences of every review are fully visible to everyone
- Rest of the review (~3/4 of the text) blurs out
- Users pay to unlock full review reading
- This is the core paywall — simple, visible, obvious value

### Monetization — FINAL (Q11)
- **$2.99/month subscription** — locked in
- Paywall: header + first 1-2 sentences visible free, rest blurs out
- "See More" button → subscription purchase page
- Without sub: you can browse, see posts exist, read teasers — but can't read full reviews
- This mirrors the TF app model (free to browse, pay to go deeper)
- 7-day free trial before first charge — standard cancel-anytime model
- Future monetization (not now): deeper dives, verified company accounts, optional non-anonymous messaging
