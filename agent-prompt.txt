## Part 2 — Backend (dailyPrompts.js)

Read the existing file first. Add these new routes:

### GET /api/daily-prompts/feed

Returns the feed for today's prompt — one entry per occupation type.

The INDUSTRIES list (same as frontend):
- general / General / emoji globe
- restaurant / Food and Restaurant / fork emoji
- retail / Retail and Sales / cart emoji  
- hvac / Trades and HVAC / wrench emoji
- office / Office and Corporate / briefcase emoji
- warehouse / Warehouse and Logistics / box emoji
- healthcare / Healthcare / hospital emoji

For each industry, call getPromptText(prompt, industry.key) and getPollOptions(prompt, industry.key).

Query results per industry:
SELECT response_value, COUNT(*) as count FROM prompt_responses WHERE prompt_date = $1 AND industry = $2 GROUP BY response_value

Query reaction counts per industry:
SELECT type, COUNT(*) as count FROM daily_prompt_reactions WHERE prompt_date = $1 AND occupation = $2 GROUP BY type

If user is authenticated, also:
- Check their response for each industry: SELECT response_value FROM prompt_responses WHERE prompt_date=$1 AND user_id=$2 AND industry=$3 LIMIT 1
- Check their reactions: SELECT type FROM daily_prompt_reactions WHERE prompt_date=$1 AND occupation=$2 AND user_id=$3

Get userOccupation from req.query.occupation or default to 'general'.

Put user's occupation post FIRST in the posts array, then the rest in order.

Set isPinned: true only on the user's occupation post.

Response shape (JSON):
{
  date: string,
  promptId: number,
  userOccupation: string,
  posts: [
    {
      occupation: string,
      occupationLabel: string,
      occupationEmoji: string,
      question: string,
      hook: string,
      responseType: string,
      userResponse: string or null,
      results: object (keyed by response value, each has count and pct),
      totalResponses: number,
      likeCount: number,
      saveCount: number,
      userLiked: boolean,
      userSaved: boolean,
      isPinned: boolean
    }
  ]
}

Build results the same way as the existing GET /today handler.

Use optionalAuth middleware.

### POST /api/daily-prompts/feed/:date/:occupation/react

Body: { type: "like" or "save" }
Use requireAuth.
Toggle the reaction: if it exists, delete it; if not, insert it.
Use uuidv4() for new reaction ids.
After toggle, query current counts for that date+occupation.
Return: { liked: boolean, saved: boolean, likeCount: number, saveCount: number }
